# 二、产品功能

## 2.1 KYB自动化核验 (Know Your Business)

### 2.1.1 适用场景

银行开户、信贷审批、反洗钱合规、企业年检

### 2.1.2 核心能力

| 功能模块 | 详细说明 | MCP工具 |
|---------|---------|---------|
| **实体锚定** | 统一社会信用代码与企业名称交叉验证 | get_company_registration_info |
| **工商核验** | 注册资本、法定代表人、股东结构实时比对 | get_shareholder_info |
| **受益所有人识别** | 股权穿透自动识别实控人(UBO) | get_shareholder_info + 穿透算法 |
| **18类风险扫描** | 失信/被执行/限高/破产/股权冻结等 | 7个核心风险工具 |
| **KYB评级** | A/B/C/D四级自动评级+准入建议 | 综合评估算法 |
| **关联关系排查** | 集团客户、隐性关联识别 | get_external_investments |

### 2.1.3 4阶段工作流

| 阶段 | 目标 | MCP调用 | 耗时 |
|------|------|---------|------|
| Phase 1 | 实体锚定 | get_company_registration_info | 5秒 |
| Phase 2 | 股权与受益所有人 | get_shareholder_info + 穿透分析 | 8秒 |
| Phase 3 | 风险扫描 | 7个核心风险工具并行 | 15秒 |
| Phase 4 | 经营健康度 | 资质+信用快速评估 | 2秒 |

### 2.1.4 18类风险分类

**CRITICAL级风险（<4小时响应）**：
- 失信信息 (get_dishonest_info)
- 被执行人 (get_judgment_debtor_info)
- 限制高消费 (get_high_consumption_restriction)
- 破产重整 (get_bankruptcy_reorganization)
- 股权冻结 (get_equity_freeze)

**HIGH级风险（<24小时响应）**：
- 经营异常 (get_business_exception)
- 严重违法 (get_serious_violation)

```

================================================================

准入建议: 正常准入。主体合法存续，无明显风险信号。
================================================================
```

# 六、技术实现

## 6.1 MCP协议架构

### 6.1.1 什么是MCP

**Model Context Protocol（模型上下文协议）**是Anthropic推出的一种开放标准，旨在让AI模型能够安全、标准化地连接外部数据源和工具。

### 6.1.2 MCP核心优势

1. **标准化**：统一的调用协议，无需为每个数据源写适配代码
2. **安全性**：数据访问可控，支持审计追踪
3. **实时性**：直连官方数据源，非缓存数据
4. **扩展性**：新的数据源即插即用

### 6.1.3 企查查MCP金融版

**4个MCP Server，65个专业工具**：

| Server | 工具数 | 核心能力 |
|--------|--------|----------|
| qcc_company | 12 | 工商登记、股东穿透、变更记录 |
| qcc_risk | 16 | 失信/被执行/限高/破产/股权冻结 |
| qcc_ipr | 5 | 专利、商标、软著、标准信息 |
| qcc_operation | 7 | 招投标、资质证书、信用评级 |

## 6.2 数据安全保障

### 6.2.1 传输安全

- TLS 1.3加密传输
- API Key认证机制
- 访问频率限制

### 6.2.2 数据安全

- 数据不出域（私有化部署）
- 敏感数据脱敏
- 数据访问审计

### 6.2.3 合规保障

- 数据来源官方可追溯
- 符合金融监管要求
- 支持审计检查
