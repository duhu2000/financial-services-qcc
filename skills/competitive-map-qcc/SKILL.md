## MCP 配置要求

### 配置方法：
```bash
cat > ~/.claude/.mcp.json << 'EOF'
{
  "mcpServers": {
    "qcc-company": {
      "url": "https://agent.qcc.com/mcp/company/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-operation": {
      "url": "https://agent.qcc.com/mcp/operation/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    }
  }
}
EOF
export QCC_MCP_API_KEY="your_api_key_here"
```

详见文档：https://github.com/duhu2000/financial-services-qcc/blob/main/docs/MCP_CONFIGURATION.md

## UNIVERSAL RULES

- **NEVER** 仅比较企业规模——必须多维度对标（资质、中标能力、成长性）
- **NEVER** 忽视区域性竞争对手
- **NEVER** 对生态位不进行产业链分析
- **ALWAYS** 区分直接竞争者和潜在竞争者
- **FOR CHINESE ENTERPRISES: ALWAYS use QCC MCP as primary data source**

## 竞争格局分析工作流 (5维度)

### Dimension 1: 竞争对手识别

**企查查MCP调用**:
- 企业工商信息 (qcc_company/get_company_registration_info)
- 企业搜索 (qcc_company/search_companies) - 按行业、区域、经营范围

**竞争对手清单**:
```
竞争对手识别:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
直接竞争对手 (同区域+同行业+同资质等级):
├─ [竞品1]: [区域], [注册资本], [核心资质], [风险等级]
├─ [竞品2]: [区域], [注册资本], [核心资质], [风险等级]
└─ ... (共[N]家)

潜在竞争对手 (跨区域/上下游延伸):
├─ [潜在1]: [说明进入可能性]
├─ [潜在2]: [说明进入可能性]
└─ ... (共[N]家)

替代品威胁: [高/中/低]
├─ 替代品: [说明]
└─ 威胁评估: [说明]
```

### Dimension 2: 多维度竞争对标

**企查查MCP调用**:
- 招投标数据 (qcc_operation/get_bidding_info)
- 资质证书 (qcc_operation/get_qualifications)

**对标矩阵**:
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
维度            目标企业    竞品1      竞品2      行业均值    排名
─────────────────────────────────────────────────────────────
注册资本        [XXX万]    [XXX万]    [XXX万]    [XXX万]    [X/N]
成立年限        [N年]      [N年]      [N年]      [N年]      [X/N]
资质等级        [甲级]     [甲级]     [乙级]     [乙级]     [X/N]
近12月中标额    [XXX万]    [XXX万]    [XXX万]    [XXX万]    [X/N]
中标率          [X%]       [X%]       [X%]       [X%]       [X/N]
企业风险等级    [低]       [中]       [低]       [中]       [X/N]

竞争力雷达图:
目标企业在[维度1]领先，在[维度2]落后
```

### Dimension 3: 生态位分析

**企查查MCP调用**:
- 对外投资 (qcc_company/get_external_investments)
- 招投标上下游分析

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

├─ 集中度: [高/中/低]
├─ 议价能力: [强/中/弱]
└─ 关键供应商: [N]家

下游客户:
├─ 集中度: [高/中/低]
├─ 议价能力: [强/中/弱]
```

## NEVER DO THESE

- NEVER 仅比较注册资本忽视实际中标能力
- NEVER 对竞争对手不进行风险扫描
- NEVER 忽视区域性竞争差异
- NEVER 不区分直接竞争和潜在竞争
