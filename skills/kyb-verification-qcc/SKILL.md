## MCP 配置要求

**⚠️ 重要：使用本SKILL前，必须确保企查查MCP服务器已配置**

### 检查清单：
1. ✅ `~/.claude/.mcp.json` 文件存在且配置正确
2. ✅ `QCC_MCP_API_KEY` 环境变量已设置
3. ✅ Claude Code 已重启加载MCP配置

### 配置方法：
```bash
# 1. 创建 MCP 配置文件
cat > ~/.claude/.mcp.json << 'EOF'
{
  "mcpServers": {
    "qcc-company": {
      "url": "https://agent.qcc.com/mcp/company/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-risk": {
      "url": "https://agent.qcc.com/mcp/risk/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    }
  }
}
EOF

# 2. 设置 API Key
export QCC_MCP_API_KEY="your_api_key_here"

# 3. 重启 Claude Code
```

详见文档：https://github.com/duhu2000/financial-services-qcc/blob/main/docs/MCP_CONFIGURATION.md

## UNIVERSAL RULES (适用于所有KYB任务)

- **NEVER** 仅凭客户提供的信息完成KYB核验——**必须**通过企查查MCP工具获取官方数据
- **NEVER** 使用网页搜索代替企查查MCP——所有中国企业数据必须通过 `qcc-company` / `qcc-risk` 等MCP工具获取
- **NEVER** 忽视注册资本实缴与认缴的差异——标注"认缴"并提示实缴情况未知
- **NEVER** 将存在股权冻结、失信、被执行的企业直接评级为"低风险"
- **ALWAYS** 明确标注数据时效性和数据来源——所有企查查数据必须标注查询时间
- **ALWAYS** 对疑似异常信息触发人工复核提示——包括但不限于：近期股权变更、地址频繁变更、经营范围重大调整
- **FOR CHINESE ENTERPRISES: ALWAYS use QCC MCP as primary data source** —— Companies House和Creditsafe对中国企业无覆盖

## KYB核验工作流 (4阶段)

### Phase 1: 实体锚定与基础信息核验 (Entity Verification)

**目标**: 确认企业主体真实存在，申请材料与官方登记信息一致

**企查查MCP调用**:
1. **企业登记信息查询** (qcc_company/get_company_registration_info)
   - 核验企业名称、统一社会信用代码是否匹配
   - 获取法定代表人、注册资本、成立日期、登记状态

2. **企业联系方式查询** (qcc_company/get_contact_info)
   - 获取工商登记电话、邮箱、官网
   - 与客户提供的联系方式比对

**核验要点**:
| 字段 | 客户提供 | 企查查官方 | 核验结果 |
|------|---------|-----------|---------|
| 企业名称 | [ ] | [ ] | [一致/不一致] |
| 统一社会信用代码 | [ ] | [ ] | [一致/不一致] |
| 法定代表人 | [ ] | [ ] | [一致/不一致] |
| 注册资本 | [ ] | [ ] | [一致/不一致] |
| 成立日期 | [ ] | [ ] | [一致/不一致] |
| 登记状态 | [ ] | [ ] | [存续/注销/吊销] |
| 注册地址 | [ ] | [ ] | [一致/不一致] |

**异常标记**:
- 🚨 **CRITICAL**: 企业已注销、吊销、列入严重违法
- ⚠️ **WARNING**: 法定代表人近期变更（6个月内）、注册地址变更
- ℹ️ **INFO**: 注册资本认缴制（非实缴）

### Phase 2: 股权结构与受益所有人识别 (UBO Identification)

**目标**: 识别实际控制人，穿透识别受益所有人

**企查查MCP调用**:
1. **股东信息查询** (qcc_company/get_shareholder_info)
   - 获取前10大股东及持股比例
   - 识别控股股东（持股>50%或相对控股）

2. **对外投资查询** (qcc_company/get_external_investments)
   - 了解企业对外投资情况
   - 识别集团关联关系

3. **主要人员查询** (qcc_company/get_key_personnel)
   - 获取董事、监事、高管名单
   - 识别关键管理人员

**受益所有人识别规则** (根据央行《关于加强反洗钱客户身份识别有关工作的通知》)：
```
受益所有人判定标准：
├─ 直接或者间接拥有25%以上股权或表决权的自然人
├─ 通过人事、财务等其他方式对公司进行控制的自然人
└─ 公司的高级管理人员（如无法识别前述两类）
```

**输出格式**:
```
股权结构分析:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
控股股东:        [股东名称] ([持股比例]%)
实际控制人:      [姓名] (通过[直接持股/间接控股/协议控制])
受益所有人:      [姓名1], [姓名2] (各持股[X]%)

股权穿透路径:
├─ [实控人姓名] → [持股平台/公司] → [目标企业] ([X]%)
└─ [其他自然人] → [直接持股] → [目标企业] ([X]%)

关联关系排查:
├─ 集团关联企业: [N]家
├─ 一致行动人:   [已识别/未识别]
└─ 隐性关联:     [需关注/无异常]
```

### Phase 2.5: 股权结构双源互证与差异归因 (Equity Cross-Verification)

**目标**: 当企业具有公开披露文件（招股说明书 / 定期报告 / 年报）时，启用"MCP + 披露文件"双源互证，避免差异误判。

**适用触发条件**（满足任一即启用）:
- 企业处于 IPO 申报 / 在审 / 已受理
- 已上市公司（A股 / H股 / 美股 / 新三板）
- 公司债 / 中票 / 短融发行人（存在募集说明书）
- 客户主动提交了招股书 / 年报 / 定期报告

---

**规则 A · 对照基准时点选取（★ 必读）**

对照披露文件里的股权结构时，**必须按以下优先级选取基准快照**：

| 优先级 | 快照类型 | 说明 |
|---|---|---|
| P1（首选）| 招股说明书**签署日**的最新披露结构 | 时间最近，与 MCP 实时数据可比性最强 |
| P2 | **最近一次股权变动完成后**的结构 | 若 P1 不可得 |
| P3 | **股改时点**结构（如 2023-07 股改基准日） | 仅在 P1/P2 缺失时使用 |
| P4 | **设立时点**结构 | 仅作历史沿革参考，**不作对照基准** |

**⚠️ 强制要求**:
- 禁止默认选择"股改时点"作为对照基准（这是常见错误）
- 当被迫使用 P3 或 P4 时，必须在差异说明栏显式标注："基准时点距今 >N 个月，差异为历史沿革，不代表数据异常"
- 招股说明书通常在"股本演变 / 股权沿革"章节按时间顺序列出**多次股权变动事件**（sec 1 设立、sec 2 增资、sec 3 股改、sec 4 转让、sec 5 最新转让……）—— 必须找到**最末一次事件后的结构**作为对照基准

---

**规则 B · 股权差异归因（★ 必读，替换掉所有启发式归因）**

**🚫 禁止**仅凭"股比变小"就归因为"转让"。这是当前模型最常见的错误，会误导法务/风控走错合规尽调路径。

**✅ 必须**按以下步骤做事件归因：

**Step 1 · 抽取完整事件序列**

从招股书"股权沿革 / 股本演变 / 股权变动"章节，按时间顺序抽取**所有股权变动事件**，每个事件必须标注事件类型：

| 事件类型 | 特征 |
|---|---|
| 设立（初始出资）| 企业成立时点的首轮注资 |
| **增资扩股（发行新股）**| 新投资者入股 / 老股东追加投资 → **导致其他未参与方被动稀释** |
| 股权转让 | 老股东之间 / 新老股东之间的股份易手 → 仅交易双方股比变动 |
| 股改（整体变更）| 有限责任公司 → 股份公司 |
| 股权激励 | 员工期权池释放 / 限制性股票授予 |
| 代持还原 | 历史代持关系解除 |
| 回购注销 | 股份回购并注销 → 剩余股东按比例被动放大 |

**Step 2 · 每个股东的差异必须映射到具体事件**

对每位股东的股比变动，回答：「哪个事件让他变了？变了多少？」—— 不得泛泛输出"小幅转让"。

**Step 3 · 一致性校验（增资 vs 转让的关键判别法）**

- 若**多个未参与任何股权转让交易**的股东，其股比按**接近相同的比例**同步下降（如陈德强 -0.23pp 陈德强_原 35.72 × 0.006 ≈ 0.21、苏州知彼 -0.08pp 11.62 × 0.007 ≈ 0.08 …… 比例都在 0.63–0.70% 区间）
  → 这是**增资扩股稀释**的数学特征
  → **必须归因为"增资稀释"**，不得归因为"转让"

- 若某股东股比下降但**没有任何其他股东相应增加**，或减幅明显大于均匀稀释率
  → 该股东**同时参与了额外的股权转让**（需定位到具体 sec X）

- 若 MCP 返回的股东备注明确写出"D+ 轮战略投资人 / Pre-IPO 轮投资者 / 员工持股平台"等字样
  → 必为**新股认购（增资）**，不是老股转让

**Step 4 · 无法归因时的降级处理**

当招股书章节信息不足或事件序列模糊时：
- **禁止**臆测事件类型
- 输出"股权变动（类型待核，建议查招股书 sec X）"
- 在最终结论中列为"❓ 待核项"，不纳入风险评级

**Step 5 · 证据链强制要求**

每条差异说明必须附具体章节定位，格式：

```
sec [N] | [事件类型] | [日期] | [事件简述]
例：sec 5 | 股权转让 | 2024-11 | 荷花缘+杨京转让至君安控股
例：sec 3 | 增资扩股 | 2024-06 | D+ 轮战略投资人中信证券入股 0.65%
```

---

**规则 C · 股东实体名称消歧（双源互证强化）**

当 MCP 股东名称与招股书股东名称不完全一致时（常见场景：控股母公司 vs 投资主体、全称 vs 简称、工商名 vs 品牌名），**不得直接判为 ❓ 单源**，必须按以下匹配规则逐级核对：

| 层级 | 匹配字段 | 命中即判 |
|---|---|---|
| L1 | `股比百分数` + `持股数量` 完全匹配 | ✅ 双源互证 |
| L2 | `工商登记地` + `USCC 前 8 位地区码` 匹配 | ✅ 双源互证（建议备注实体差异说明）|
| L3 | `控股关系`（如 MCP："中信证券投资有限公司" vs 招股书："中信投资" → 核查是否为中信证券控股子公司）| ✅ 双源互证 |
| L4 | 三层都对不上 | ❓ 单源 |

**MCP 返回的股东备注字段**（如"D+ 轮战略投资人"、"持股平台"、"员工激励平台"）**必须作为证据链组成部分**进入差异说明。

---

**规则 D · 差异说明输出模板（标准化）**

所有股权差异必须按以下格式输出，**不得自由发挥**：

```
| 股东名称 | MCP 当前 | 招股书对照时点 | 差异说明 | 证据链 | 互证状态 |
|---------|----------|--------------|---------|-------|---------|
| 陈德强  | 35.49%   | 35.49%（签署日快照）| 无差异 | sec 5 末态 | ✅ 双源一致 |
| XX 基金 | 0.62%    | 0.65%（D+ 轮认购）  | 增资稀释后微调 | sec 3 增资 | ✅ 双源互证 |
```

**不允许出现**的差异说明（历史错误示范，供模型警戒）：
- ❌ "2024 年小幅转让"（未定位事件 sec 号、未区分增资/转让）
- ❌ "时点差"（未指明是哪个时点、距今多久）
- ❌ "股权变动"（未说明事件类型）

### Phase 4: 经营健康度与持续经营能力 (Operational Health)

**目标**: 评估企业持续经营能力，识别空壳公司、僵尸企业

**企查查MCP调用** (qcc_operation Server):
1. **资质证书查询** (get_qualifications)
   - 核查行业特许经营资质
   - 检查资质有效期

2. **行政许可查询** (get_administrative_license)
   - 核查经营许可有效性
   - 识别超范围经营风险

3. **招投标信息查询** (get_bidding_info)
   - 评估业务活跃度
   - 识别业务断崖式下跌

4. **抽查检查记录** (get_spot_check_info)
   - 核查监管检查结果
   - 识别质量问题

5. **新闻舆情查询** (get_news_sentiment)
   - 监控负面舆情
   - 识别声誉风险

**经营健康度评估**:
```
持续经营能力评估:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
经营活跃度:    [高/中/低/无业务] (近12个月招投标[N]次)

├─ A级: 正常经营，无风险信号
├─ B级: 轻微异常，可正常准入
├─ C级: 中高风险，需加强尽调
```

## NEVER DO THESE

- NEVER 仅凭客户提供的信息完成KYB核验——必须与官方数据交叉验证
- NEVER 对存在失信、被执行记录的企业直接评级为"低风险"
- NEVER 忽视注册资本认缴与实缴的差异——必须明确标注
- NEVER 对空壳公司（无实际经营、无员工、无社保）正常准入
- NEVER 跳过受益所有人识别——这是反洗钱合规的强制要求
- NEVER 忽视集团客户关联关系——可能导致集中度风险
- NEVER 对高风险客户不设置后续监测频率
- **NEVER 默认取"股改时点"作为股权对照基准**——必须优先取签署日最新快照（见 Phase 2.5 规则 A）
- **NEVER 仅凭"股比变小"就归因为"股权转让"**——必须做增资/转让事件类型判别（见 Phase 2.5 规则 B）
- **NEVER 在多个非交易方按相同比例同步稀释时归因为转让**——这是增资扩股的教科书特征，误归因会导致法务走错尽调路径
- **NEVER 因股东全称 vs 简称差异直接判为"单源"**——必须先做股比+股数+控股关系三级消歧（见 Phase 2.5 规则 C）
- **NEVER 输出"2024 年小幅转让 / 时点差 / 股权变动"这类无证据链的差异说明**——每条差异必须定位到招股书 sec N 和具体事件
