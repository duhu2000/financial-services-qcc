# 合规风险持续监控 (Compliance Monitor) v1.0

## 技能概述

这个Skill用于**7x24小时持续监控企业合规风险**，专为银行合规部、券商风控部门、投资机构投后管理团队设计。

**核心价值**：
- **实时监控**：企查查MCP自动扫描18类风险信号
- **智能预警**：风险事件实时推送，分级预警
- **投后管理**：被投企业/贷款客户风险持续跟踪
- **合规报告**：自动生成日报/周报/月报

**适用场景**：
- 银行贷后风险监控
- 投资机构投后管理
- 券商客户风险监测
- 供应链风险预警
- 合规审计支持

---

## 企查查MCP增强说明

### 18类风险实时监控

企查查`qcc-risk` Server覆盖的18类风险信号：

| 风险类别 | 监控内容 | 预警级别 |
|----------|----------|----------|
| **司法风险** | 失信记录、被执行人、限高消费、终本案件 | 🔴 高 |
| **经营风险** | 经营异常、严重违法、行政处罚、吊销 | 🔴 高 |
| **财务风险** | 欠税公告、税收违法、股权冻结、股权质押 | 🟡 中 |
| **清算风险** | 破产重整、清算信息、简易注销 | 🔴 高 |
| **执行风险** | 被执行人、失信被执行人、限制出境 | 🔴 高 |

### 监控频率

- **实时预警**：高风险信号（失信、被执行）T+1推送
- **每日扫描**：全量18类风险每日自动扫描
- **定期报告**：支持日报/周报/月报自动生成

## 标准工作流

### 第一步：添加监控对象

输入企业名称列表，Skill自动：
1. 识别企业全称和统一信用代码
2. 建立监控档案
3. 获取初始风险快照

### 第二步：7x24小时监控

```
企查查MCP监控循环：
┌─────────────────────────────────────────┐
│           监控对象池                     │
│    [企业A] [企业B] [企业C] ...          │
└──────────────┬──────────────────────────┘
               │
        ┌──────▼──────┐
        │ 每日定时扫描 │
        │  qcc-risk   │
        └──────┬──────┘
               │
    ┌──────────┼──────────┐
    │          │          │
┌───▼───┐ ┌───▼───┐ ┌───▼───┐
│风险变化│ │新增风险│ │风险解除│
└───┬───┘ └───┬───┘ └───┬───┘
    │         │         │
    └─────────┴─────────┘
              │
         ┌────▼────┐
         │预警推送  │
         │报告生成  │
         └─────────┘
```

### 第三步：风险预警推送

当监控企业出现风险变化时：
- 🔴 高风险：立即推送（失信、被执行、限高）
- 🟡 中风险：每日汇总（股权冻结、行政处罚）
- 🟢 风险解除：每日汇总（移出异常、执行完毕）

### 第四步：生成监控报告

支持多种报告周期：
- **日报**：前一日风险变化汇总
- **周报**：本周风险趋势分析
- **月报**：月度风险监控总结

## 风险评分模型

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 司法风险 | 35% | 失信、被执行、限高、诉讼 |
| 经营风险 | 25% | 异常、违法、处罚、吊销 |
| 财务风险 | 20% | 欠税、冻结、质押、抵押 |
| 清算风险 | 20% | 破产、清算、注销 |

### 风险等级

| 等级 | 分数 | 说明 | 响应时效 |
|------|------|------|----------|
| 🔴 高风险 | 80-100 | 存在严重风险信号 | 立即 |
| 🟡 中风险 | 50-79 | 存在一般风险信号 | 24小时 |
| 🟢 低风险 | 0-49 | 无风险或轻微风险 | 常规监控 |

## 使用示例

### 示例1：银行贷后监控

**用户输入**：
> "对我行贷款客户启动风险监控，名单：北京XX科技、上海XX贸易、深圳XX制造"

**Skill执行**：
1. 添加3家企业到监控池
2. 获取初始风险快照
3. 启动7x24小时监控
4. 输出监控仪表盘

**输出**：
- 监控对象：3家企业已添加
- 初始风险：1家低风险，2家中风险
- 监控状态：已启动
- 预警设置：高风险即时推送

### 示例2：投后风险预警

**用户输入**：
> "查看我被投企业的最新风险情况"

**Skill执行**：
1. 扫描被投企业风险变化
2. 对比昨日数据
3. 生成风险报告

**输出**：
- ⚠️ 发现1家新增被执行人（标的1250万）
- ✅ 1家经营异常已解除
- 📊 总体风险可控
