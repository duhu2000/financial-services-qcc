  适用场景：战略投资并购、产业整合、上市公司并购、PE/VC并购投资。

  使用方式：/merger-screening-qcc 收购方企业 [--industry 目标行业] [--scale 规模区间] [--region 区域]

## MCP 配置要求

### 配置方法：
```bash
cat > ~/.claude/.mcp.json << 'EOF'
{
  "mcpServers": {
    "qcc-company": {
      "url": "https://agent.qcc.com/mcp/company/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-risk": {
      "url": "https://agent.qcc.com/mcp/risk/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-ipr": {
      "url": "https://agent.qcc.com/mcp/ipr/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-operation": {
      "url": "https://agent.qcc.com/mcp/operation/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    }
  }
}
EOF
export QCC_MCP_API_KEY="your_api_key_here"
```

## UNIVERSAL RULES

- **NEVER** 仅基于规模匹配推荐标的——必须考虑战略协同
- **NEVER** 忽视并购后的整合风险
- **NEVER** 对标的企业不进行风险穿透核查
- **ALWAYS** 评估股权结构对交易的影响
- **FOR CHINESE ENTERPRISES: ALWAYS use QCC MCP as primary data source**

## 并购筛查工作流 (5维度)

### Dimension 1: 标的筛查

**企查查MCP调用**:
- 企业搜索 (qcc_company/search_companies)
- 风险扫描 (qcc_risk Server)

**标的长名单**:

## NEVER DO THESE

- NEVER 推荐存在重大司法风险的并购标的
- NEVER 忽视股权质押对交易的影响
- NEVER 对整合风险不进行量化评估
- NEVER 不区分控股与参股的估值差异
