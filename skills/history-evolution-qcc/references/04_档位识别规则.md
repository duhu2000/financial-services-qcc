# 档位识别规则 · V1.0

## 目标

通过前置 15c（3 个 5c 工具）的最低成本，将目标企业归入"小微 / 融资 / 上市"三档之一，从而决定后续工具调用清单与章节启用规则。

## 决策树

```
[开始]
   │
   ▼
调用 1: mcp__qcc-company__get_company_profile
   │   ├── cost=5c（累计 5c）
   │   └── 获取：注册资本、人员规模、成立日期、行业、经营状态
   ▼
调用 2: mcp__qcc-company__get_listing_info
   │   ├── cost=5c（累计 10c）
   │   └── 获取：上市状态、交易所、代码、发债记录
   ▼
调用 3: mcp__qcc-operation__get_financing_records
   │   ├── cost=5c（累计 15c）
   │   └── 获取：融资轮次、金额、投资方、日期
   ▼
【判定】
   │
   ├── 上市 或 有债券发行记录 ──→【上市档】（预算封顶 100c）
   │
   ├── 有融资记录（天使 / A / B / C / D / Pre-IPO 等任一） ──→【融资档】（预算 50–80c）
   │
   └── 无上市 且 无融资记录
       │
       ├── 注册资本 ≤ 1000 万 或 人员规模 ≤ 50 人 ──→【小微档】（预算 15–30c）
       │
       └── 注册资本 > 1000 万 且 人员规模 > 50 人 ──→【融资档】（按融资档处理，预算 50–80c）
```

## 档位特征表

| 档位 | 典型特征 | 预算 | 启用的条件扩展子节 |
|---|---|---|---|
| **小微档** | 注册资本 ≤ 1000 万、无上市、无融资、人员 ≤ 50 人 | 15–30c（¥1.5–3） | 仅 §5.6（有处罚/担保记录时） |
| **融资档** | 有融资轮次（天使—Pre-IPO），或 规模较大但未上市 | 50–80c（¥5–8） | §4.4、§5.4、§5.5、§5.6 |
| **上市档** | 已上市（A/港/美股）或 有公募债 | 封顶 100c（¥10） | §4.4、§4.6、§5.4、§5.5、§5.6、§7 财务 |

## 特殊情形处理

### 情形 1：国有独资/国资控股

不单独分档，按上述规则归入融资或上市档。国资背景不影响档位，但在 §4 渲染时追加"国有属性"标签。

### 情形 2：被投资企业是合伙企业 / 持股平台 / SPV

档位识别仍按主体本身特征判断。在股东侧穿透时如遇到持股平台（如"XX投资中心（有限合伙）"、"XX投资有限公司"），自动追加二级/三级穿透（最多 3 层），不计入档位。

### 情形 3：历史上市但已退市

按"已退市" + 当前是否有债券在存续判断：
- 有存续债券 → 上市档
- 无存续债券 → 融资档

### 情形 4：新三板挂牌

- 创新层 / 精选层 → 上市档
- 基础层 → 融资档

### 情形 5：注销 / 吊销状态

档位识别照常进行，但 §7 总结需在首段明确标注"经营状态：注销/吊销"。

## 档位升级触发

档位识别默认按上述树一次性判定。以下情形允许**运行时升级档位**（会增加成本）：

| 触发器 | 升级方向 | 依据 |
|---|---|---|
| §4 穿透时发现"国有控股"、"全资子公司 of 上市公司" | 小微 → 融资 | 母公司档位继承 |
| §5 经营轨迹发现"重大行政许可"或"高新企业认定" | 小微 → 融资 | 成熟度信号 |
| §5 风险记录发现"行政处罚 > 50 万"或"被列入严重违法失信名单" | 任意档 → 触发扩展调查（仍在本档位预算内优先消耗） | 风险合规 |

## 成本统计点

档位识别固定消耗 15c，无论结果如何。这 15c 包含在各档位预算之内（小微预算 15–30c 即 15c 识别 + 0–15c 核心调用）。

## 代码调用

```python
from scripts.tier_detector import detect_tier

tier, profile, cost = detect_tier(company_keyword="企查查科技股份有限公司")
# tier ∈ {"micro", "financing", "listed"}
# profile 是已调用的 3 个工具合并结果
# cost = 15（除非调用失败）
```

## 注意事项

1. 档位识别的 3 个工具不缓存——即使 30 分钟前查过同一家，也会重新调用（保证数据新鲜度）
2. 档位判定后不回溯：后续如果发现判断偏差，**只允许升级不允许降级**（避免报告生成中途缩水）
3. 如 3 个工具中任一返回失败（网络 / 限流），默认降级为"小微档"并在报告中标注"档位识别受限"
