### A3. 差距汇总与建议处置
- **必补（高优先，信贷/IPO 双线必需）**：1、2、3、4、5、6、9、10、11、14、18 之"严格披露"
- **新增独立小节（7 章保持，内部加子项）**：
  - §4 新增 4.4 实控人认定与一致行动 / 4.5 股权激励 / 4.6 申报前 12 个月新增股东专项
  - §5 新增 5.4 董监高与治理（现 §5.2 已有"治理变迁"描述，扩为"董事会构成 + 审计委/监事会 + 独立董事 + 职工董事 + 高管兼职"）
  - §5 新增 5.5 关联方与关联交易（6 类关联方 + 亲属关系 + 关联交易汇总）
  - §5 新增 5.6 独立性五维 + 同业竞争
  - §6 其他重要事项：处罚 / 对外担保 / 资金占用 / 重大合同 / 重要事件
- **按档位可选（§7 总结前的可选模块，"有则展示、无则跳过"）**：
  - 上市档：锁定减持承诺、稳定股价、欺诈发行回购、填补即期回报、特别表决权/VIE
  - 融资档：对赌/回购/业绩承诺、优先清算权、反稀释、私募基金备案、股权激励
  - 小微档：税务异常、行政许可到期、经营异常名录（qcc-risk 即可）

**结论**：当前 7 章框架在**信贷小微**场景是合格的；在**上市尽调 / 对标招股书**场景下缺口超过 50%。建议保留 7 章，把上述新增内容作为**章节内子项（2 级标题）**塞进，避免章节数膨胀。

- 避免整章留空；且总结章中不得因为某小节跳过而空悬

## Part C · 三方数据融合架构设计

> 输入三源 → AI 统一推理 → 输出可审计报告

### C1. 三源定义（必须通过"数据来源标签"区分，不可混写）

| 来源 | 标签 | 典型数据 | 时效性 | 法定效力 |
|---|---|---|---|---|
| MCP 实时 | `M` | 工商登记、股东穿透、风险、IPR、经营 | 秒级~日级 | 公开登记信息 |
| 机构内部 | `I` | 信贷台账、受托支付记录、授信历史、客户 KYC | 月级~日级 | 行内数据，对内具效力 |
| 客户经理现场 | `F` | 访谈纪要、内部章程、代持协议原件、对赌协议、合伙协议、非公开股东会议纪要 | 一次性 | 尽调现场披露，需签字存档 |

### C2. 字段级数据溯源（在现有 §11 行级溯源基础上扩展）

原格式：
```
[来源: tool · record_id=<ID> · snapshot=YYYY-MM-DD]
```

升级为：
```
[来源: <M|I|F> · <tool or system> · record_id=<ID> · snapshot=YYYY-MM-DD · author=<客户经理工号>]
```

例：
- `[来源: M · qcc-company/get_shareholder_info · record_id=QCC20260420-ABC · snapshot=2026-04-20]`
- `[来源: I · 客户关系系统 · record_id=CUST-44872 · snapshot=2026-04-15]`
- `[来源: F · 现场访谈 · record_id=FIELD-20260418-001 · author=L03421 · snapshot=2026-04-18]`

### C3. 三源对撞规则（Conflict Resolution）

| 情形 | 判定规则 | 报告表现 |
|---|---|---|
| M/I 数据一致 | 正常融合 | 不额外标注 |
| M/I 数据不一致，M 是登记信息 | M 优先；I 作为备注标红 | `[MCP 登记: X] vs [机构内部: Y] → 以 MCP 为准` |
| M/I 一致但 F 不同 | F 作为"未登记事项" | 单独小节"未登记事项披露"，不改写主表 |
| F 披露代持/对赌等 M 看不到 | F 成为主要信息源，M 提供公开对照 | 加入 §4.2 / §4.3 并在来源栏注明 F |
| 三源全不一致 | 暂停写入，创建`数据对撞工单`；报告留空并警示 | 标注"⚠️ 三源矛盾，已挂单 #XXX" |

### C5. 融合推理闭环（AI 推理执行顺序）

```
1. 工单初始化（实体 ID + 报告类型 + 档位自动识别）
2. 并行拉取 M（所有 qcc-* MCP）+ I（机构内部查询）+ F（现场资料导入）
3. 字段归一（时间锚点 + 字段名标准化）
4. 对撞检测（C3 规则）
5. 三源一致 → 融合写入 / 不一致 → 按规则分层
6. 章节组装（按档位可见性矩阵）
7. §11 行级溯源 + C2 升级标签
8. §7 整体评价（规则判词，M/I/F 全部计入证据链）
9. 导出 docx/pdf + 审计底稿（原始 M/I/F 记录全部归档）
```
