## 上市档（封顶 100c，¥10）

| 优先级 | 工具 | 成本 | 启用条件 | §章节用途 |
|---|---|---|---|---|
| P1 | `mcp__qcc-operation__get_financial_key_indicators` | 3c | A 股 + 发债 | §7 财务摘要 |
| P1 | `mcp__qcc-company__get_listing_info`（深查） | 5c | 已上市 | §7 |
| P1 | `mcp__qcc-operation__get_company_announcement` | 5c | 总是 | §5 / §7 |
| P2 | `mcp__qcc-executive__get_personnel_beneficial_owner` | 5c × N | §4.4 / §5.5 深化 | 关键人受益人反查 |
| P2 | `mcp__qcc-executive__get_personnel_relevant_companies`（V1.8 上线） | 5c × N | §5.5 网络深化 | 亲属/同学/共事关系 |
| P2 | `mcp__qcc-history__get_historical_admin_license` | 5c | 有许可变更 | §5.2 许可续期轨迹 |
| P3 | `mcp__qcc-risk__get_dishonest_info` / `get_exit_restriction` / `get_high_consumption_restriction` | 5c 各 | 查到有记录 | §5.6 深度风险 |
| P3 | `mcp__qcc-risk__get_chattel_mortgage_info` | 5c | 查到有记录 | §5.6 |
| P3 | `mcp__qcc-ipr__get_trademark_info` / `get_patent_info`（可选） | 5c 各 | 报告明确需要 IP 摘要时 | §7（可选附注） |

**上市档封顶保护**（由 `cost_counter.py` 自动执行）：
- 累计达 95c：仅允许调用 ≤5c 工具
- 累计达 100c：停止一切 MCP 调用，剩余字段标注「本次未采集（成本封顶）」

## 工具调用顺序建议（实现在 `mcp_orchestrator.py`）

```
[档位识别 15c]
   ↓
[必调工具批 ~15c]  company_registration_info + change_records + shareholder_info
   ↓
[按档位追加]
   ├── 小微档：业务异常 / 税务异常（按返回优先级触发）
   ├── 融资档：historical_shareholders (20c) + actual_controller + 关键人 executive
   └── 上市档：financial_indicators + announcement + 关键人扩展 + 风险深查
   ↓
[条件扩展子节]
   ├── §4.4 ← get_actual_controller 返回非空
   ├── §4.6 ← 上市档 / Pre-IPO 且 historical_shareholders 12 个月内有新增
   ├── §5.4 ← 融资/上市档 且 key_personnel 非空
   ├── §5.5 ← 融资/上市档 且 personnel_controlled_companies 非空
   ├── §5.6 ← 任一 qcc-risk 工具返回非空
   └── §7 财务 ← 上市档 且 financial_key_indicators 非空
   ↓
[build_docx 组装]
```

## 成本节约技巧

1. **批量并行**：同档位的独立工具可以并发调用（Claude Agent 原生支持同 message 多 tool calls）
2. **短路返回**：qcc-risk 类工具返回空时，不启动对应 §5.6 段落渲染
3. **关键人筛选**：`get_key_personnel` 返回列表，`personnel_positions` 等只对前 3 位（法人 + 董事长 + 总经理）展开，避免对 10+ 位全部展开
4. **历史股东按需**：小微档即使遇到代持嫌疑也不调 20c 的 `historical_shareholders`——V1.0 不做代持还原，归 V2.0

## 工具未上线时的降级策略

对 `qcc-history` / `qcc-executive` 规划中的工具：

- 若返回 "tool_not_available" / 404：章节按"MCP 未上线" 标注，不启用对应子节
- V1.0 SKILL 不 fallback 到其他数据源（保持 MCP 纯度）
- 客户侧若强烈需要历史数据，应等待产品上线或转入 V2.0 自维护
