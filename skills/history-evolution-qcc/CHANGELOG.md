## [1.1.3] - 2026-04-24

### 背景

用户在 V1.1.2 报告尾段 "本报告数据源" 发现 body 文字出现**垂直黑条/断裂线**——经定位是 `italic=True` 在 LibreOffice 对中英混排 CJK 字符渲染时产生的字形断裂。同时发现 footer 版本字符串仍写着 V1.0.2（未随版本升级）。

### 修复

- **移除 body 文字斜体**：全量 14 处 `italic=True` 改为 `italic=False`，彻底消除 LibreOffice 渲染中英混排斜体时的垂直黑条
- **版本字符串动态化**：footer 中 "V1.0.2" → "V1.1.3"，`main()` argparse description 同步更新
- 源码头部注释也已更新

### 保留

- V1.1.2 的全部样式改造（H1/H2 蓝色竖条、表格蓝底白字表头、交替条纹行）
- `_add_para()` 的 `italic` 参数仍保留（用于 source_tag 的灰色小字），只是主体文字不再使用

### 受影响范围

主要是 footer "本报告数据源" 段 + 所有章节的副标题说明段。修复前后可视化对比：

| 文件 | V1.1.2 | V1.1.3 |
|---|---|---|
| footer body | 中英字符间出现垂直黑条（字形断裂） | 干净无黑条 |
| "V1.0.2" 字串 | 错误残留 | 更新为 "V1.1.3" |

## [1.1.2] - 2026-04-24

### 背景

用户反馈 V1.1.1 报告样式未严格对齐 kyb-opus：章节标题/表格表头/数据行配色与原报告不一致；`§0` 标题 "加「数」前行，赋能发展" 是企查查 SLogan，不通用。本版针对性修复。

### 修改

- **去除企查查特定标语**：`§0` 章节标题 "加「数」前行，赋能发展" → 改为通用 **"发展里程碑"**
- **章节标题 (H1/H2) 样式严格对齐 kyb-opus**：
  - 移除 docx 默认 `Heading 1` 样式
  - H1：**左侧 4.5pt 蓝色粗竖条** + 16pt 蓝色粗字
  - H2：**左侧 3pt 蓝色竖条** + 13pt 蓝色粗字
  - H3：11pt 蓝色粗字（无竖条）
- **表格样式严格对齐 kyb-opus**：
  - 移除 `Light Grid Accent 1` docx 预置样式
  - **表头行**：实心蓝色背景 `#2563EB` + 白色粗体 10pt
  - **数据行**：浅蓝 `#F6F8FB` / 白 交替条纹（斑马条纹）
  - **边框**：浅灰 `#E5E7EB` 细边框（不突兀）
- **新增样式工具函数**（`_set_cell_bg`, `_set_cell_border`, `_set_para_left_bar`, `_set_shading`）——基于 docx 底层 `OxmlElement` 直接注入 XML

### 保留

- V1.1.1 章节结构：§0 / §1-§7 / §8 / §附录
- V1.1.1 渲染逻辑：圆点+竖线时间轴 + 表格
- V1.1.0 弃用的 4 个曲线/柱/折线/卡片函数仍保留在 `build_timeline.py` 中（供 V2.0 自定义）

### 样本对比

| 版本 | 视觉特征 |
|---|---|
| V1.1.1 | docx 默认 Heading 1（灰蓝细线）+ `Light Grid Accent 1` 表格（灰白交替，无蓝底表头） |
| V1.1.2 | **kyb-opus 风格**：蓝色竖条章节 + 蓝底白字表头 + 浅蓝/白条纹行 |

## [1.1.0] - 2026-04-24

对齐 QCC 官方 2025 公司介绍 PPT（P5 里程碑 / P8 荣誉资质 / P11 技术 / P12 产品生态）的立体叙事范式，将 V1.0.2 的"纯工商时间轴"升级为**立体发展历程**：成立 → 发展 → 壮大全程串联，覆盖员工规模、办公布局、证书资质、知识产权、业务扩张、行业地位、荣誉榜单 等 10+ 维度。

### 新增

- **§0 里程碑总览（PPT P5 风格）**：置于 §1 公司概况之后
  - 淡蓝渐变上升曲线背景 + 年份圆球锚点 + 事件标题+副标签
  - 自动从 `registration / financing / honor / listing` 多源聚合里程碑
  - 筛选规则：成立 / 改制 / 融资节点 / 国家级认证 / 重大荣誉 / 上市挂牌
  - 单张 PNG（160 DPI，宽 17cm），嵌入文档顶部
- **§8 发展综览**：置于 §7 总结之后
  - **§8.1 许可与资质**：qualifications 数据 + 有效证书 Top 8
  - **§8.2 知识产权布局**：专利+商标+软件著作权+作品著作权+互联网备案 数量卡片 + 专利年度分布条形图
  - **§8.3 业务扩张与团队规模**：员工规模年度趋势折线图（从年报社保人数抽）+ 分支机构 + 对外投资 Top 10
  - **§8.4 行业地位与荣誉**：荣誉/上榜/国家级数量 4 卡片 + 重点荣誉表 + 近期上榜 Top 8
  - **§8.5 官方信用评级**（条件启用）
- **`scripts/build_timeline.py` 新增 3 个渲染函数**：
  - `render_milestone_curve()` — PPT P5 上升曲线
  - `render_stat_cards()` — PPT P8 大数字卡片
  - `render_staff_trend()` — 员工规模年度折线图
  - `render_ipr_yearly_bars()` — 知识产权年度条形图
- **Orchestrator 新增 14 个工具**（按档位启用）：
  - 小微档 +2：honor_info / qualifications
  - 融资档 +10：honor / ranking_list / qualifications / telecom_license / recruitment / credit_evaluation / patent / trademark / software_copyright / annual_reports
  - 上市档 +14：融资档全部 + copyright_work / internet_service / news_sentiment / import_export_credit

### 档位预算调整

- 小微档：30c → **45c**（+15c 新工具缓冲）
- 融资档：80c → **130c**（+50c，10 个新 5c 工具）
- 上市档：100c → **180c**（+80c，14 个新工具）

### V2.0 客户自升级指引追加

### 背景

对齐 QCC Advanced Report（China mainland KYC 报告）的 CHANGE HISTORY 章节视觉范式 —— 历史数据从表格升级为真正的时间轴（圆点+竖线+日期锚定），提升 KYB 报告的专业度和阅读效率。

  - BytesIO 输出，无临时文件

- **Advanced Report 配色锁定**：

  - 已知日期实心 ● 蓝点，起点未知空心 ○ 灰点

### 修改

- **§2 注册资本 / §3 名称 / §6 地址 / §4.3 股东进出 / §5.4 历任董监高 / §附录 大事年表**：全部从表格改为嵌入时间轴 PNG

- **依赖**：`scripts/requirements.txt` 新增 `matplotlib>=3.6.0`

### 保留不变

- §1 公司概况（字段表格）
- §4.1 当前股东（表格，股份型 vs 有限公司型动态切换）
- §4.4 实控人（表格）
- §5.2 分支 / 对外投资 / 行政许可（表格）
- §5.5 关联方（表格 + B3 Top-N 截断）
- §5.6 处罚与担保（表格）
- §7 财务摘要（3 年对比表，仅上市档启用）

### 字体依赖说明

- **macOS**：系统自带 PingFang SC，无需安装
- **Linux**：`sudo apt install fonts-noto-cjk`（推荐）或 `fonts-droid-fallback`
- **Windows**：系统自带 Microsoft YaHei

如果 SKILL 找不到任何 CJK 字体，时间轴里的中文会显示为方块；降级方案是用户自行回退到 V1.0.1（纯表格版）。

### 已知限制

- matplotlib 的字体混排用了手动 per-char 拆分，ASCII/CJK 交界处可能有微小间距差异（视觉上可以忽略）
- DPI 固定 160；若需更高清晰度打印可修改 `build_timeline._render_to_bytes()` 中的 `dpi` 参数

- **§7 财务摘要改为 3 年对比**：主要财务指标/利润表/资产负债表/现金流量表/盈利能力/偿还能力/成长能力 7 类按 3 年横向对比
- **3 档真实样本**：

- **sample_snapshot_micro.json / sample_snapshot_financing.json**：小微档、融资档真实快照

## [1.0.0] - 2026-04-20

### 定位

首个正式版。纯企查查 MCP 驱动，零客户材料负担。

### 新增

- **SKILL.md 主入口**：5 步工作流（输入收集 → 档位识别 → MCP 批量调用 → 7 章组装 → DOCX/PDF 输出）
- **7 章报告模板**：公司概况 / 注册资本 / 名称变更 / 股权演变 / 经营轨迹 / 地址变迁 / 总结
- **三档自适应**：上市 / 融资 / 小微档识别与差异化输出
- **6 个条件扩展子节**（档位达标自动启用，不达标整节不生成）
  - §4.4 实控人认定
  - §4.6 申报前 12 月新增股东
  - §5.4 董监高与治理
  - §5.5 关联方网络（公开可查部分）
  - §5.6 处罚与担保
  - §7 财务摘要（仅 A 股 + 发债）
- **执行脚本**：
  - `tier_detector.py` — 15c 档位识别
  - `cost_counter.py` — 100c 封顶保护
  - `mcp_orchestrator.py` — 按档位批量调用 MCP
  - `build_docx.py` — 7 章 DOCX/PDF 生成
- **参考文档**（references/）：方案文档、报告模板、字段映射、档位规则、工具清单、V2.0 升级指引、升级建议留档
- **示例产物**：企查查科技股份有限公司示例报告（DOCX + PDF）

### 数据源覆盖

- 67 个现有 MCP 工具（qcc-company 14 / qcc-risk 34 / qcc-ipr 6 / qcc-operation 14）
- 70 个规划 MCP 工具（qcc-history 34 / qcc-executive 35 规划 Phase 1–3）
- 1 个财务工具（`get_financial_key_indicators`，A 股 + 发债）
- 合计 138 个工具

### 覆盖率

- 小微档：100%（MCP 全覆盖）
- 融资档：90%（仅对赌/回购类非公示合同条款不覆盖，归 V2.0）
- 上市档：82%（代持、一致行动、独立性五维、股权激励细则、关联交易明细等 5 项归 V2.0）

### 明确剔除（归 V2.0 客户自升级）

1. 代持还原、一致行动、独立性五维、股权激励细则、关联交易明细
2. 对赌 / 回购 / 业绩承诺 / 优先清算
3. 同业竞争承诺、股利分配政策、上市承诺体系
4. 特别表决权 / AB 股 / VIE 架构
5. 亲属关系定性网络图

### 限制

- 财务摘要仅 A 股上市 + 发债企业可用
