## MCP 配置要求

**⚠️ 重要：使用本SKILL前，必须确保企查查MCP服务器已配置**

### 检查清单：
1. ✅ `~/.claude/.mcp.json` 文件存在且配置正确
2. ✅ `QCC_MCP_API_KEY` 环境变量已设置
3. ✅ Claude Code 已重启加载MCP配置

### 配置方法：
```bash
# 1. 创建 MCP 配置文件
cat > ~/.claude/.mcp.json << 'EOF'
{
  "mcpServers": {
    "qcc-company": {
      "url": "https://mcp.qcc.com/data/company/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-risk": {
      "url": "https://mcp.qcc.com/data/risk/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-ipr": {
      "url": "https://mcp.qcc.com/data/ipr/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-operation": {
      "url": "https://mcp.qcc.com/data/operation/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    }
  }
}
EOF

# 2. 设置 API Key
export QCC_MCP_API_KEY="your_api_key_here"

# 3. 重启 Claude Code
```

详见文档：https://github.com/duhu2000/financial-services-qcc/blob/main/docs/MCP_CONFIGURATION.md

## UNIVERSAL RULES (适用于所有IC Memo任务)

- **NEVER** 仅凭目标企业提供的材料完成尽调——**必须**通过企查查MCP工具获取官方数据
- **NEVER** 使用网页搜索代替企查查MCP——所有中国企业数据必须通过 qcc-company/qcc-risk/qcc-ipr/qcc-operation MCP工具获取
- **NEVER** 忽视知识产权的权利状态——必须核查专利是否有效、商标是否被撤销
- **NEVER** 对存在重大司法风险的企业直接给出"建议投资"结论
- **ALWAYS** 明确标注数据来源和时间——所有企查查数据必须标注查询时点
- **ALWAYS** 对估值敏感性因素进行风险提示——包括但不限于：股权纠纷、核心技术权属不明
- **FOR CHINESE TARGETS: ALWAYS use QCC MCP as primary data source** —— 这是唯一权威数据源

## IC MEMO 生成工作流

### Step 1: 输入收集 (Input Gathering)
```
用户输入: /ic-memo-qcc [目标企业名称] [--round Series B] [--sector 半导体]

必需信息:
├─ 目标企业名称（全称或关键词）
├─ 拟投轮次（用于确定尽调深度）
└─ 所属行业（用于选择可比公司）

可选信息:
├─ 已知融资金额/估值
├─ 主要竞争对手
└─ 特殊关注事项
```

### Step 2: 数据自动采集 (Automated Data Collection)
```
并行调用企查查MCP四大Server:

Phase 2.1: 企业基础信息 (qcc_company) [预估5秒]
Phase 2.2: 知识产权扫描 (qcc_ipr) [预估5秒]
Phase 2.3: 司法风险扫描 (qcc_risk) [预估8秒]
Phase 2.4: 经营与舆情 (qcc_operation) [预估5秒]

Total API Calls: [15-20个]
Total Time: < 30秒
```

### Step 3: 智能分析 (AI Analysis)
```
├─ 股权穿透与受益所有人识别
├─ 知识产权价值评估
├─ 司法风险定级与影响评估
├─ 经营健康度评分
└─ 行业对标与估值分析
```

### Step 4: 报告生成 (Report Generation)
```
生成标准IC Memo格式 (Markdown)
包含7个章节 + 执行摘要
自动填充数据到对应章节
生成风险清单与投资建议
```

### Step 5: 输出与导出 (Output)
```
输出格式:
├─ Markdown (可编辑)
├─ PDF (可打印提交投委会)
└─ 关键数据Excel (财务/股权/风险清单)
```

## NEVER DO THESE

- NEVER 仅凭企业提供的BP和财务预测完成IC Memo——必须独立验证
- NEVER 忽视知识产权的权利瑕疵——可能导致投资后核心技术无法使用
- NEVER 对存在股权代持、VIE架构瑕疵的企业直接给出投资建议
- NEVER 忽略实控人的其他关联企业风险——可能引发关联交易或资金占用
- NEVER 在IC Memo中模糊处理关键风险——必须明确、量化、可追踪
- NEVER 对存在重大诉讼、执行记录的企业不做风险缓释建议
- NEVER 跳过竞对分析——可能导致估值严重偏离市场

ALL OUTPUTS REQUIRE REVIEW BY INVESTMENT COMMITTEE AND LEGAL COUNSEL BEFORE INVESTMENT DECISION.
