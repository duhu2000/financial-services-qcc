## MCP 配置要求

**⚠️ 重要：使用本SKILL前，必须确保企查查MCP服务器已配置**

### 检查清单：
1. ✅ `~/.claude/.mcp.json` 文件存在且配置正确
2. ✅ `QCC_MCP_API_KEY` 环境变量已设置
3. ✅ Claude Code 已重启加载MCP配置

### 配置方法：
```bash
# 1. 创建 MCP 配置文件
cat > ~/.claude/.mcp.json << 'EOF'
{
  "mcpServers": {
    "qcc-risk": {
      "url": "https://agent.qcc.com/mcp/risk/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }

    "qcc-company": {
      "url": "https://agent.qcc.com/mcp/company/stream",

  }
}
EOF

# 2. 设置 API Key
export QCC_MCP_API_KEY="your_api_key_here"

# 3. 重启 Claude Code
```

## UNIVERSAL RULES (适用于所有诉讼分析任务)

- **NEVER** 仅统计案件数量——必须分析案件性质、金额、结果
- **NEVER** 忽视案件金额与注册资本的比例——大额诉讼可能致命
- **NEVER** 对未结案件不进行风险提示
- **NEVER** 忽略作为原告的诉讼——可能反映应收账款管理问题
- **ALWAYS** 区分已结案件与未结案件的风险权重
- **ALWAYS** 分析诉讼趋势（上升/下降/集中爆发）
- **ALWAYS** 评估执行风险（被执行人、失信、终本案件）
- **FOR CHINESE ENTERPRISES: ALWAYS use QCC MCP as primary data source**

## 诉讼分析工作流 (5维度深度分析)

### Dimension 1: 涉诉全景统计 (Litigation Overview)

**目标**: 全面盘点企业涉诉情况

**企查查MCP调用**:
1. **裁判文书查询** (qcc_risk/get_court_judgments)
   - 全部涉诉案件清单
   - 判决结果与金额

2. **开庭公告查询** (qcc_risk/get_court_announcements)
   - 近期开庭案件
   - 案件进展跟踪

3. **立案信息查询** (qcc_risk/get_case_filing_info)
   - 已立案未判决案件

**涉诉统计表**:
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
角色        案件数    涉案金额(万元)    占比
─────────────────────────────────────────
作为被告    [N]件    [XXX]万元        [X]%
作为原告    [N]件    [XXX]万元        [X]%

诉讼频率:    [每X个月1起]
涉诉率:      [N]% (与同行业对比[高/中/低])
```

2. **失信被执行人查询** (qcc_risk/get_dishonest_info)
   - 失信记录详情
   - 失信原因

3. **终本案件查询** (qcc_risk/get_terminated_cases)

## NEVER DO THESE

- NEVER 仅统计案件数量而不分析性质与金额
- NEVER 对作为被告的大额未结诉讼轻描淡写
- NEVER 忽视被执行人的当前状态（是否已履行）
- NEVER 对群体性诉讼不特别标注
- NEVER 不计算涉诉金额与注册资本的比例
- NEVER 忽略诉讼趋势的判断（上升/下降）
- NEVER 对涉诉企业不给出明确的投资建议
- NEVER 不标注数据查询时间——诉讼情况动态变化

ALL OUTPUTS REQUIRE REVIEW BY A QUALIFIED LEGAL COUNSEL BEFORE USE IN INVESTMENT OR BUSINESS DECISIONS.
