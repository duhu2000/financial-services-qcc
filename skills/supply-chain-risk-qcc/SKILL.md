## MCP 配置要求

**⚠️ 重要：使用本SKILL前，必须确保企查查MCP服务器已配置**

### 检查清单：
1. ✅ `~/.claude/.mcp.json` 文件存在且配置正确
2. ✅ `QCC_MCP_API_KEY` 环境变量已设置
3. ✅ Claude Code 已重启加载MCP配置

### 配置方法：
```bash
# 1. 创建 MCP 配置文件
cat > ~/.claude/.mcp.json << 'EOF'
{
  "mcpServers": {
    "qcc-operation": {
      "url": "https://agent.qcc.com/mcp/operation/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-risk": {
      "url": "https://agent.qcc.com/mcp/risk/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    },
    "qcc-company": {
      "url": "https://agent.qcc.com/mcp/company/stream",
      "headers": { "Authorization": "Bearer ${QCC_MCP_API_KEY}" }
    }
  }
}
EOF

# 2. 设置 API Key
export QCC_MCP_API_KEY="your_api_key_here"

# 3. 重启 Claude Code
```

详见文档：https://github.com/duhu2000/financial-services-qcc/blob/main/docs/MCP_CONFIGURATION.md

## UNIVERSAL RULES (适用于所有供应链风险评估)

- **NEVER** 仅评估一级供应商——必须考虑二级、三级供应商风险传导
- **NEVER** 忽视单一供应商依赖风险——集中度是核心风险指标
- **NEVER** 对供应商的司法风险不进行穿透核查
- **NEVER** 忽略地理集中度风险——区域集中可能受自然灾害/政策影响
- **ALWAYS** 评估供应商的替代性——关键物料是否有备选来源
- **ALWAYS** 分析企业在供应链中的议价能力
- **ALWAYS** 标注供应链数据的时效性
- **FOR CHINESE ENTERPRISES: ALWAYS use QCC MCP as primary data source**

## 供应链风险评估工作流 (5维度)

### Dimension 1: 供应商全景扫描 (Supplier Overview)

**目标**: 盘点企业供应链结构

**企查查MCP调用**:
1. **招投标信息** (qcc_operation/get_bidding_info)
   - 通过中标/投标信息识别主要供应商/客户

2. **企业对外投资** (qcc_company/get_external_investments)
   - 识别关联供应商/客户

**供应商统计**:
```
供应链结构概览:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

二级供应商    [N]家      [N]家         [高/中/低]
三级供应商    [N]家      [N]家         [高/中/低]

├─ 原材料供应商: [N]家
├─ 零部件供应商: [N]家
├─ 外协加工商:   [N]家
├─ 物流服务商:   [N]家

```

### Dimension 2: 关键供应商风险评估 (Key Supplier Risk)

**目标**: 深度评估关键供应商风险

**企查查MCP调用**:
1. **工商信息** (qcc_company/get_company_registration_info)
   - 供应商基本情况

2. **司法风险** (qcc_risk/get_dishonest_info, get_judgment_debtor_info)
   - 供应商涉诉、被执行情况

3. **经营异常** (qcc_risk/get_business_exception)
   - 供应商经营状态

**关键供应商风险表**:

## NEVER DO THESE

- NEVER 仅评估一级供应商而忽视多级传导风险
- NEVER 对单一供应商依赖不标注风险
- NEVER 忽视供应商的司法风险
- NEVER 对区域集中度风险不进行提示
- NEVER 对关键物料的替代性不进行评估
- NEVER 忽视供应商的经营异常信号
- NEVER 对休眠供应商不提示风险
- NEVER 不评估供应链的应急响应能力
